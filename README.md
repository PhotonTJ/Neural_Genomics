## Overview

**nDNA (Neural DNA)** is a semantic-genotypic representation that captures the *latent identity* of AI foundation models through the intrinsic *geometry of belief*. Rather than evaluating models by their outputs alone, nDNA reads their internal cognition — treating language models as **semantic organisms** with traceable ancestry, mutation, and inheritance.

The framework synthesizes three principled dimensions of latent geometry:

| Dimension | Mathematical Origin | What It Reveals |
| :--- | :--- | :--- |
| **Spectral Curvature** (κ<sub>ℓ</sub>) | Riemannian manifolds | Curvature of conceptual flow across layers |
| **Thermodynamic Length** (ℒ<sub>ℓ</sub>) | Statistical thermodynamics | Semantic effort of representational transitions |
| **Belief Gradients** (‖**v**<sub>ℓ</sub><sup>(c)</sup>‖) | Semantic vector dynamics | Torsion fields guiding belief directional orientation |

These converge into a composite **nDNA fingerprint** — a coordinate-free, physics-grade readout of how meaning is *bent*, *paid for*, and *pushed* through a model's layers.

## nDNA Metrics — Quick Reference

| Metric | Symbol | Description |
| :--- | :--- | :--- |
| Parameter-space effort | E<sub>ℓ</sub> | Per-layer squared-gradient norm — how hard a layer works relative to its parameter count |
| Thermodynamic length | Δ<sub>ℓ</sub> | Fisher–Rao distance between consecutive prediction distributions |
| Alignment | α<sub>ℓ</sub> | Theoretical vs. actual belief update direction |
| Belief push norm | ‖v̄<sub>ℓ</sub>‖ | Magnitude of belief change per layer |
| Spectral curvature | κ<sub>ℓ</sub> | Curvature of the prediction manifold along teacher-forced token paths |
| Composite nDNA | nDNA<sub>pred</sub> | Scalar + layerwise curves combining κ, Δ, and belief norms |
| Eta | η<sub>ℓ</sub> | Δ²/E — FR energy normalized by effort (collapse diagnostic) |

## Model Zoo & Adapter Configuration

<details>
<summary>Expand model zoo and adapter details</summary>
<br/>

- `model_zoo.json` holds alias → HF id mappings plus an `enabled` flag. Method-5 generic resolves models from this file unless `--hf-models` is provided.
- Collapse driver reads the zoo and runs only entries with `enabled: true` unless filtered by `--model-keys`.
- LoRA runner defaults to the Qwen3-4B base model and region-specific adapters in `nDNA/Qwen3-4B-WikiCulture-SFT`. Merged adapters are expected under `<repo>/<merged_subdir>/<pair>` where pair is `REGION_REGION` (e.g., `AF_EU`).
- Fisher merging scripts require LoRA adapters with identical configuration (target modules, rank, alpha, dropout, RSLoRA/DoRA flags). The merge utility enforces config equality before combining weights.

## Detailed Script Reference

<details>
<summary><strong>Method-5 Generic Runner</strong> — full specification</summary>
<br/>

**Script:** `scripts/run_method5_generic.py`

**Purpose:** Evaluate one or many HF models on one or many datasets using Method-5 geometry. Supports streaming datasets, optional curvature + nDNA, and MXFP4 quantization.

**Input selection:** Resolve models from `model_zoo.json` (enabled entries) unless `--hf-models` or `--model-keys` override. Dataset keys: `squad`, `squad_v2`, `stanford_plato`, `ag_news`, `hh_rlhf`, `gsm8k`, `mbpp`, `alpaca`, `everything_instruct_multilingual`, `wikipedia`, `automathtext`, plus legacy aliases.

**Outputs:** `.npz` named `<dataset>__method5_<model>.npz` in `--out-dir`. Contains: model/dataset tags, E_l, n_examples, n_params, Delta, Alpha, Vnorm, mean_total_fr, n_tokens, belief_norms, tau, max_len, batch_size, tokens_per_ex, fisher_unit, optional kappa/kappa_positions, optional ndna_scalar/ndna_layerwise.

**Compute order:** E_l → FR Δ/α/vnorm → belief norms → optional kappa/nDNA.

**Failure modes:** missing pad token, no enabled models in zoo, unknown dataset key, empty dataset after filtering, kappa shape mismatch (skips nDNA_pred), MXFP4 with old transformers.

```bash
# Minimal single model/dataset
python -m scripts.run_method5_generic --dataset alpaca \
  --hf-models meta-llama/Meta-Llama-3-8B \
  --max-samples 64 --max-len 256 --batch-size 2 --tokens-per-ex 16

# All enabled zoo models on multiple datasets
python -m scripts.run_method5_generic --datasets ag_news hh_rlhf gsm8k \
  --max-samples 128 --max-len 192 --batch-size 1 --tokens-per-ex 8 --fisher-unit token

# Enable curvature/nDNA, force streaming
python -m scripts.run_method5_generic \
  --datasets everything_instruct_multilingual wikipedia \
  --streaming --max-samples 64 --max-len 256 --batch-size 1 \
  --tokens-per-ex 12 --tau 0.7 --kappa-keep-last-k 12 --kappa-include-embedding-node
```
</details>

<details>
<summary><strong>Method-5 LoRA Runner</strong> — full specification</summary>
<br/>

**Script:** `scripts/run_method5_lora.py`

**Purpose:** Run Method-5 on a base model with PEFT adapters (fine-tuned regional adapters and Fisher-merged combos). Ensures adapters are attached with `is_trainable=True` for Fisher gradients.

**Adapter resolution:** `--adapters` selects `finetuned`, `merged`, or `all`. `--adapter-names` overrides explicit subset. `--adapter-repo` defaults to `nDNA/Qwen3-4B-WikiCulture-SFT`.

**Outputs:** `<dataset>__method5_lora_<adapter_type>_<adapter>.npz` with FR metrics, belief norms, optional E_l, optional kappa/nDNA, metadata on base model and adapter path/type.

**Failure modes:** adapter config mismatch, missing pad token, no adapters resolved, invalid dataset key, gradient disabled on params.

```bash
# All fine-tuned adapters on Alpaca with Fisher + FR
python -m scripts.run_method5_lora --adapters finetuned --dataset alpaca \
  --max-samples 64 --max-len 192 --tokens-per-ex 12 --tau 0.8 --fisher-unit sequence

# Merged adapters only, skip Fisher for speed
python -m scripts.run_method5_lora --adapters merged --dataset ag_news \
  --max-samples 96 --skip-param-effort --no-kappa --max-len 256

# Mixed selection with local adapter dirs and MXFP4
python -m scripts.run_method5_lora --adapter-names AF EU AF_EU \
  --local-adapters-dir ./adapters --local-merged-dir ./merged_adapters \
  --dataset hh_rlhf --use-mxfp4 --skip-param-effort
```
</details>

<details>
<summary><strong>Collapse Driver</strong> — full specification</summary>
<br/>

**Script:** `scripts/run_collapse_from_zoo.py`

**Purpose:** Automate cross-breeding or inbreeding collapse trajectories across multiple models. Each generation runs Alpaca Method-5 and spectral curvature.

**Protocols:** `--protocol cross` fine-tunes on human Alpaca pairs; `--protocol inbreed` generates synthetic Alpaca-style data with the current model and fine-tunes on it.

**Run layout:** `<base-run-dir>/<model_key>/<run_name>/gen0` holds base geometry; each subsequent `genX` contains `model/` (optional) and `metrics/{method5_unified.json, spectral_curvature.json}`.

**Geometry config:** `geom_max_samples`, `geom_max_len`, `geom_batch_size`, `geom_unit`, `spectral_prompts`, `spectral_max_tokens`. Smoke mode (`--smoke`) lowers sample counts for quick validation.

**Failure modes:** empty model selection, missing metrics files for plotting, vLLM unavailable for inbreeding (falls back to HF generation), insufficient GPU memory.

```bash
# Cross-breeding 3 generations with full geometry
python -m scripts.run_collapse_from_zoo \
  --protocol cross --model-keys llama3-8b \
  --base-run-dir ./collapse_runs_multi --run-name cross_a \
  --generations 3 --max-train-samples 6000 \
  --max-steps-first 1000 --max-steps-later 500 \
  --geom-max-samples 256 --geom-max-len 256 --geom-batch-size 2 --save-models

# Inbreeding with smoke settings
python -m scripts.run_collapse_from_zoo \
  --protocol inbreed --model-keys gpt2 --smoke --run-name quick_inbreed
```
</details>

<details>
<summary><strong>Fisher Computation & Merging</strong> — full specification</summary>
<br/>

**Fisher script:** `ndna_lib/merging/compute_fisher_lora.py` — computes diagonal empirical Fisher for trainable LoRA parameters using WikiCulture region-filtered text. Saves safetensors containing squared gradients averaged per example.

**Regions:** WikiCulture configs named by `REGION` (AF, AS, AU, CH, EU, LA, ME, NA). Iterator filters `bucket_geo` to requested region.

**Merge script:** `ndna_lib/merging/fisher_merge_lora.py` — Fisher-weighted merge of multiple LoRA adapters. Requires identical adapter configurations.

```bash
# Compute Fisher for a single region
python -m ndna_lib.merging.compute_fisher_lora \
  --base_model Qwen/Qwen3-4B \
  --adapter_dir ./adapters/AF \
  --dataset_id nDNA/WikiCulture --region AF \
  --streaming --max_examples 256 --batch_size 1 --max_length 512 \
  --dtype bf16 --shuffle_buffer 8000 --out_path ./fishers/AF.safetensors

# Fisher-merge two adapters with equal weights
python -m ndna_lib.merging.fisher_merge_lora \
  --adapters ./adapters/AF ./adapters/EU \
  --fishers ./fishers/AF.safetensors ./fishers/EU.safetensors \
  --normalize_fishers mean --out_dir ./merged_adapters/AF_EU
```
</details>

<summary><strong>Alignment (SFT & DPO)</strong> — full specification</summary>
<br/>

**Scripts:** `llama3p1_IT.py`, `qwen2p5_IT.py`, `ministral3_IT.py` (SFT); `llama3p1_DPO.py`, `qwen2p5_DPO.py`, `ministral3_DPO.py` (DPO).

**Typical flow:** Run IT script → instruction-following adapter → merge with `merge.py` → Run DPO script using IT checkpoint as `ref_model` on preference datasets.

<details>
<summary><strong>Knowledge Distillation</strong> — full specification</summary>
<br/>

**Scripts:** `distill_math_lora.py`, `llama_distillation.py`, `qwen_distillation.py`.

**Purpose:** Transfer specific reasoning capabilities from a larger teacher model into a student model. Trains LoRA adapters on the student while teacher is frozen.

**Loss:** Combination of supervised CE (on answer tokens) and KL divergence (using teacher's logits). Controlled by `--kl_weight`, `--ce_weight`, `--temperature`.

```bash
python distillation/llama_distillation.py \
  --dataset_name gsm8k --dataset_config main \
  --train_split train --eval_split test \
  --output_dir distillation/llama3-math-distill \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 16 \
  --learning_rate 2e-4 --num_train_epochs 1 \
  --kl_weight 0.7 --ce_weight 0.3 --temperature 2.0 --bf16 True
```
</details>

## Troubleshooting

<details>
<summary>Expand troubleshooting checklist</summary>
<br/>

- Verify GPU: `torch.cuda.is_available()` — CPU is supported but slow; collapse training requires CUDA
- **Padding errors**: ensure tokenizer has `pad_token` or `eos_token`; runners set pad to EOS when missing
- **Streaming datasets**: use reasonable `--shuffle-buffer-size` (10k default) to avoid order bias
- **MXFP4 errors**: use `--no-mxfp4` with older transformers versions; MXFP4 cannot be combined with E_l (Fisher)
- **Fisher computation**: adapters must load with `is_trainable=True`; gradients must be forced on floating parameters
- **vLLM unavailable**: inbreeding falls back to standard generation (slower)
- **Plotting skips**: check console for "Skipping ... missing keys" messages
- **DPO training collapse**: requires correct SFT checkpoint as `ref_model`; wrong checkpoint → zero KL
- **Distillation OOM**: use `--use_4bit` when teacher + student exceed GPU memory
- **SFT special tokens**: scripts fix zero-initialized special token embeddings before training; do not skip this for new base models
- **Concreteness POS**: install with `python -m spacy download en_core_web_sm`
- **Empty datasets**: ensure `--max-samples` > 0; empty materialized dataset after filtering will cause errors
- **Kappa shape mismatch**: if kappa shape doesn't match Δ, nDNA_pred computation is skipped automatically
</details>
