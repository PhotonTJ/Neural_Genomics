## Overview

**nDNA (Neural DNA)** is a semantic-genotypic representation that captures the *latent identity* of AI foundation models through the intrinsic *geometry of belief*. Rather than evaluating models by their outputs alone, nDNA reads their internal cognition — treating language models as **semantic organisms** with traceable ancestry, mutation, and inheritance.

The framework synthesizes three principled dimensions of latent geometry:

| Dimension | Mathematical Origin | What It Reveals |
| :--- | :--- | :--- |
| **Spectral Curvature** ($\kappa_\ell$) | Riemannian manifolds | Curvature of conceptual flow across layers |
| **Thermodynamic Length** ($\mathcal{L}_\ell$) | Statistical thermodynamics | Semantic effort of representational transitions |
| **Belief Gradients** ($\lVert\mathbf{v}_\ell^{(c)}\rVert$) | Semantic vector dynamics | Torsion fields guiding belief directional orientation |

These converge into a composite **nDNA fingerprint** — a coordinate-free, physics-grade readout of how meaning is *bent*, *paid for*, and *pushed* through a model's layers.

## nDNA Metrics — Quick Reference

| Metric | Symbol | Description |
| :--- | :--- | :--- |
| Parameter-space effort | $E_\ell$ | Per-layer squared-gradient norm — how hard a layer works relative to its parameter count |
| Thermodynamic length | $\Delta_\ell$ | Fisher–Rao distance between consecutive prediction distributions |
| Alignment | $\alpha_\ell$ | Theoretical vs. actual belief update direction |
| Belief push norm | $\lVert\bar{v}_\ell\rVert$ | Magnitude of belief change per layer |
| Spectral curvature | $\kappa_\ell$ | Curvature of the prediction manifold along teacher-forced token paths |
| Composite nDNA | $\text{nDNA}_\text{pred}$ | Scalar + layerwise curves combining κ, Δ, and belief norms |
| Eta | $\eta_\ell$ | $\Delta^2/E$ — FR energy normalized by effort (collapse diagnostic) |

## 🛠 Toolkit — Getting Started

### Environment Setup

```bash
# Make the setup script executable
chmod +x scripts/setup_env.sh

# Run the setup (installs all dependencies, configures auth for GitHub & HuggingFace)
source scripts/setup_env.sh

# Activate the environment in future sessions
conda activate ndna
```

### Requirements

- Python 3.9+ with dependencies in `requirements.txt`
- GPU strongly recommended (bfloat16 default; Fisher and FR paths rely on autograd)
- `HF_TOKEN` for gated models/datasets
- Outputs: compressed `.npz` for Method-5, JSON for collapse/concreteness, Plotly HTML for dashboards

## Command Inventory

<details>
<summary><strong>Method-5 Generic Runner</strong> — <code>scripts/run_method5_generic.py</code></summary>
<br/>

Evaluate one or many HF models on one or many datasets using Method-5 geometry. Supports streaming, optional curvature + nDNA, and MXFP4 quantization.

```bash
# Single model, multiple datasets
python -m scripts.run_method5_generic \
  --hf-models meta-llama/Meta-Llama-3-8B \
  --datasets ag_news hh_rlhf gsm8k \
  --max-samples 96 --max-len 192 --batch-size 1 \
  --tokens-per-ex 12 --tau 0.8 --fisher-unit token

# With curvature/nDNA, streaming
python -m scripts.run_method5_generic \
  --dataset everything_instruct_multilingual \
  --streaming --max-samples 64 --shuffle-buffer-size 20000 \
  --max-len 256 --batch-size 1 --tokens-per-ex 12 \
  --tau 0.7 --kappa-keep-last-k 12 --kappa-include-embedding-node
```
</details>

## End-to-End Playbooks

<details>
<summary><strong>Method-5 Generic Sweep</strong></summary>
<br/>

1. Configure `model_zoo.json` with enabled models
2. Run: `python -m scripts.run_method5_generic --datasets ag_news hh_rlhf gsm8k`
3. Collect `.npz` under `results/method5_generic/<dataset>/`
4. Visualize: `python -m scripts.plot --input-dir results/method5_generic/<dataset>`
</details>

<details>
<summary><strong>LoRA Fisher Pipeline</strong></summary>
<br/>

1. Compute Fishers per region with `ndna_lib.merging.compute_fisher_lora`
2. Merge adapters with `ndna_lib.merging.fisher_merge_lora`
3. Evaluate: `python -m scripts.run_method5_lora --adapters merged`
4. Plot with `scripts/plot.py` or cross-dataset overlays
</details>

<details>
<summary><strong>Collapse Experiment</strong></summary>
<br/>

1. Run: `python -m scripts.run_collapse_from_zoo --protocol cross --model-keys llama3-8b`
2. Visualize: `python -m scripts.plot_collapse_pairs_3d --base_dir <run_dir> --belief_key Eta`
</details>

<details>
<summary><strong>Full Alignment Pipeline (SFT → DPO → Distillation)</strong></summary>
<br/>

1. **SFT**: Run `Alignment/llama3p1_IT.py` → produces instruction-following adapter
2. **Merge**: `python Alignment/merge.py` → merged model for DPO
3. **DPO**: Run `Alignment/llama3p1_DPO.py` with IT checkpoint as `ref_model`
4. **Distillation**: `python distillation/llama_distillation.py --kl_weight 0.7 --ce_weight 0.3`
</details>

## Troubleshooting

<details>
<summary>Expand troubleshooting checklist</summary>
<br/>

- Verify GPU: `torch.cuda.is_available()` — CPU is supported but slow; collapse training requires CUDA
- **Padding errors**: ensure tokenizer has `pad_token` or `eos_token`; runners set pad to EOS when missing
- **Streaming datasets**: use reasonable `--shuffle-buffer-size` (10k default) to avoid order bias
- **MXFP4 errors**: use `--no-mxfp4` with older transformers versions
- **Fisher computation**: adapters must load with `is_trainable=True`
- **vLLM unavailable**: inbreeding falls back to standard generation (slower)
- **Plotting skips**: check console for "Skipping ... missing keys" messages
- **DPO training collapse**: requires correct SFT checkpoint as `ref_model`; wrong checkpoint → zero KL
- **Distillation OOM**: use `--use_4bit` when teacher + student exceed GPU memory
- **spaCy POS**: install with `python -m spacy download en_core_web_sm`
</details>
